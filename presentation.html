ARIMA Step 2

ADF 單根檢定

# Prompt 對這組收盤價資料做 ADF 單根檢定， 告訴我 test statistic、p-value，以及結論（是否為定態序列）

預期結果

p-value > 0.05
→ 無法拒絕單根假設
→ 非定態

解讀

原始股價序列通常是非定態的
需要做差分處理
這就是 ARIMA 中 d 的來源

ARIMA Step 4

模型估計

36 / 55

ARIMA Step 5

殘差診斷 — 最容易被忽略的步驟

# Prompt 對殘差做 Ljung-Box 檢定（lag=10）和 Jarque-Bera 常態性檢定， 畫殘差的 ACF 圖，判斷殘差是否為白噪音

Ljung-Box

檢驗殘差是否有自相關
p-value > 0.05 → OK

Jarque-Bera

檢驗殘差是否常態分配
財務資料通常不通過

如果殘差不是白噪音，你的模型就是有問題的 — 很多人跑完模型就直接看結果，不檢查殘差

GARCH

從均值到波動度

ARIMA 預測價格的走向
GARCH 分析價格的波動幅度

在財務領域，我們更關心的往往是「波動度」

波動群聚 Volatility Clustering

報酬率不是均勻散佈的：

  • 有些時期波動很大（COVID-19）
  • 有些時期相對平靜
  • 大波動之後往往跟著大波動

這就是我們需要 GARCH 的原因

41 / 55

GARCH Step 1-2

報酬率時序圖 + ARCH-LM 檢定

# Step 1：觀察波動群聚 計算台股加權指數的日報酬率（對數報酬率）， 畫出報酬率的時序圖，觀察是否有波動群聚現象
# Step 2：ARCH 效果檢定 對報酬率做 ARCH-LM 檢定（lag=5）， 告訴我 p-value 和結論：是否存在 ARCH 效果

如果 p-value < 0.05 → 存在 ARCH 效果 → 波動度會隨時間變化 → 需要 GARCH

GARCH Step 3

GARCH(1,1) 估計

# Prompt 估計 GARCH(1,1) 模型， 顯示 omega、alpha、beta 係數，以及 alpha + beta 的值

係數意義

omega常數項（長期波動度）
alphaARCH 項（短期衝擊）
betaGARCH 項（波動持續性）

關鍵指標

alpha + beta ≈ 1

接近 1 = 高持續性
一旦市場劇烈波動
會持續好一陣子才平息

VAR Step 3

Granger 因果檢定

# Prompt 做 Granger 因果檢定，檢驗： 1. 匯率是否 Granger-cause 股市報酬 2. 利率是否 Granger-cause 股市報酬 3. 股市報酬是否 Granger-cause 匯率 顯示 F-statistic 和 p-value

重要觀念

Granger 因果 ≠ 真正因果

它只是說：一個變數的過去值能不能幫助預測另一個變數

在時間序列分析裡，這已經是很有用的資訊

47 / 55

VAR Step 4

衝擊反應函數 IRF

# Prompt 畫出衝擊反應函數（IRF）， 觀察「匯率衝擊」對「股市報酬」的影響持續多久， 包含 95% 信賴區間，期間設定 20 期

匯率貶值一個標準差後，股市反應多久？多少期回到零？

48 / 55

VAR Step 5

變異數分解 FEVD

# Prompt 做預測誤差變異數分解（FEVD），期間 20 期， 看股市報酬的預測誤差中，有多少比例來自匯率和利率的衝擊

量化影響力

如果匯率解釋了股市報酬變異的 15%

→ 匯率對股市有實質的影響力

比「有影響」更有說服力

FEVD 給你具體數字

不只是說「有影響」

而是告訴你「影響有多大」
