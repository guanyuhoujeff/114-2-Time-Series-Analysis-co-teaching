### 1-2. 論文架構巡禮（12min）

**【轉場】**

> 好，在聊 AI 工具之前，我們先快速過一遍：一篇學術論文長什麼樣子？
>
> 在座的應該都讀過不少論文了，但你們有沒有認真想過——這些章節到底在幹嘛？為什麼要這樣安排？

**【重點提示】**

- **Introduction**：回答「為什麼要做這個研究？」
  - 研究背景 → 重要性 → 文獻不足 → 研究目的
  - 像是論文的「電梯簡報」——30 秒內說服讀者你的研究值得看

- **Literature Review**：回答「前人做了什麼？」
  - 不是文獻的堆疊（A 說 XXX，B 說 YYY），而是有邏輯的整理
  - 最後要收斂到「所以目前的缺口是什麼」→ 帶出你的研究問題

- **Methodology**：回答「你怎麼做的？」
  - 讓別人能重現你的研究
  - 要說明為什麼選這個方法（不是只列公式）

- **Results**：回答「你發現了什麼？」
  - 圖表為主、文字為輔
  - 客觀呈現，不要在這裡加太多解讀

- **Conclusion**：回答「So what？」
  - 研究貢獻、實務意涵、研究限制、未來方向

**【怎麼想題目 — 逐字稿】**

> 接下來一個博士生最常問的問題：題目怎麼來的？
>
> 大部分好題目是「讀出來的」，不是「想出來的」。

**【重點提示】**

- **題目生成的流程**：
  1. 從你的興趣和指導教授的專長出發
  2. 大量閱讀近 3-5 年的文獻
  3. 找到 research gap（文獻缺口）
  4. 從缺口提出研究問題
  5. 研究問題 → 假說 → 方法 → 資料

- **好題目的四個標準**：有學術價值、有資料可做、你有能力做、三年內做得完

- **時間序列相關範例**：
  - 「GARCH 模型在加密貨幣市場的波動度預測」
  - 「COVID-19 對台股波動結構的衝擊——門檻 VAR 模型分析」
  - 「美國聯準會升息對亞洲匯率的傳導效果——VECM 分析」

### 1-3. 文獻搜尋全攻略（25min）

**【轉場】**

> 找文獻這件事，十年前只有一個方法——Google Scholar 加上圖書館資料庫。現在 AI 文獻工具大爆發，多到你不知道該用哪個。
>
> 今天我不是要介紹每個工具的功能，而是要教你們一套**工作流程**。

**【一條龍工作流程 — 逐字稿】**

（投影片展示流程圖）

> **Step 1：Elicit — 用問題搜文獻**
>
> Elicit 跟 Google Scholar 最大的差別是：你不是打關鍵字，你是打一個研究問題。比如你打「How does GARCH model perform in cryptocurrency volatility forecasting?」，它會回傳相關文獻，而且自動幫你提取每篇的研究方法、樣本、主要發現。
>
> 免費版就可以無限搜尋和看摘要，非常夠用。

（打開 Elicit 示範搜尋）

> **Step 2：Semantic Scholar — 看引用關係**
>
> 找到幾篇重要的文獻之後，你要看它們的「引用網路」。Semantic Scholar 完全免費。
>
> 它的 TLDR 功能——每篇論文用一句話摘要。你快速掃過 20 篇的 TLDR，就知道哪幾篇值得仔細讀。
>
> 時間序列領域特別適合用引用追蹤。比如從 Engle (1982) 的 ARCH 原始論文出發，往後追 cited by，你會看到整個 GARCH 家族的演化脈絡。

（打開 Semantic Scholar 示範）

> **Step 3：ScienceDirect — 下載全文**
>
> 前兩步找到了文獻，但你只看到摘要。學校有訂閱 ScienceDirect，在校內網路可以直接下載 PDF。

> **Step 4：SciSpace — 用 AI 讀論文**
>
> 有些論文真的很難讀——尤其是數學推導很多的那種。SciSpace 讓你把 PDF 上傳，直接用中文問問題。
>
> 比如上傳一篇 GARCH-MIDAS 的論文，問「這篇論文的 MIDAS 部分是怎麼設定的？低頻變數用了什麼？」，它會從論文裡找答案。

（打開 SciSpace 示範上傳 PDF）

> **Step 5：Research Rabbit — 擴展閱讀清單**
>
> 你已經有了幾篇核心文獻，Research Rabbit 像文獻版的 Spotify——給它幾篇種子論文，它推薦你可能感興趣的其他文獻。

**【傳統方法不能丟 — 重點提示】**

- **Google Scholar**：最廣的搜尋範圍
- **JCR / Impact Factor**：判斷期刊品質
- **引用追蹤法**：往前追 reference、往後追 cited by——最笨但最可靠
- AI 工具是加速器，不是替代品。最後你還是要自己讀論文。

**【小結】**

> **Elicit 找 → Semantic Scholar 追 → ScienceDirect 下載 → SciSpace 讀 → Research Rabbit 擴展**。記住這個流程。


（執行，展示 ACF/PACF）

**【Step 4：模型估計】**

```
用 ARIMA(p,d,q) 估計模型（用你剛才建議的參數），
顯示係數估計值、標準誤、p-value 和 AIC
```

（執行）

**【Step 5：殘差診斷】**

```
對殘差做 Ljung-Box 檢定（lag=10）和 Jarque-Bera 常態性檢定，

```

（執行）

> 殘差診斷是最容易被忽略的步驟。很多人跑完模型就直接看結果，不檢查殘差。如果殘差不是白噪音，你的模型就是有問題的。

**【Step 6：預測】**

```

畫出預測圖，包含 95% 信賴區間
```

（執行，展示預測圖）

> 六步完成一個完整的 ARIMA 分析。你們在 EViews 裡做這些大概要 20-30 分鐘，這裡大概 5 分鐘。

## 第 2 節（50min）— GARCH 與 VAR 模型實作

---

### 2-1. GARCH 模型（25min）

**【轉場】**

> 好，ARIMA 處理的是「均值方程式」——預測價格的走向。但在財務領域，我們更關心的往往是「波動度」——價格的變動幅度。
>
> 你們第 5-6 週學過 ARCH/GARCH。今天用 Vibe Coding 來跑。

**【Step 1：觀察波動群聚】**

```
計算台股加權指數的日報酬率（對數報酬率），
畫出報酬率的時序圖，觀察是否有波動群聚現象
```

（執行）

> 看到了嗎？報酬率不是均勻散佈的——有些時期波動很大（例如 2020 年初 COVID-19），有些時期相對平靜。這就是波動群聚（Volatility Clustering）。

**【Step 2：ARCH 效果檢定】**

```
對報酬率做 ARCH-LM 檢定（lag=5），
告訴我 p-value 和結論：是否存在 ARCH 效果
```

（執行）

> 如果 p-value 很小（< 0.05），代表存在 ARCH 效果——波動度會隨時間變化。這就是我們需要 GARCH 的原因。

**【Step 3：GARCH(1,1) 估計】**

```
估計 GARCH(1,1) 模型，
顯示 omega、alpha、beta 係數，以及 alpha + beta 的值
```

（執行）

> 重點看 alpha + beta。如果接近 1，代表波動度有很高的持續性——一旦市場開始劇烈波動，會持續好一陣子才平息。這跟你們課本上講的一致。

**【Step 4：條件波動率視覺化】**

```
畫出條件波動率（conditional volatility）的時序圖，
標註 2020 年 COVID-19 和 2022 年台海危機的時間點
```

（執行，展示條件波動率圖）

> 看到了嗎？COVID-19 那段時間的條件波動率明顯飆高。這張圖可以直接放進你的論文 Results 裡。

### 2-2. VAR 模型（25min）

**【轉場】**

> 最後一個模型——VAR。你們第 11 週會學到，今天先用 Vibe Coding 預習一下。
>
> ARIMA 和 GARCH 都是看「一個變數」，VAR 是看「多個變數之間的互動關係」。比如：股市報酬、匯率、利率之間會互相影響嗎？

**【Step 1：多變量資料準備】**

```
請準備三組資料（近 5 年日資料）：
1. 台股加權指數日報酬率
2. 美元兌台幣匯率日變動率
3. 台灣 10 年期公債殖利率日變動率
畫出三組資料的時序圖
```

（執行）

**【Step 2：VAR 模型估計】**

```
用 AIC 準則選擇最適落後期數，估計 VAR 模型，
顯示選出的 lag 數和各方程式的係數摘要
```

（執行）

**【Step 3：Granger 因果檢定】**

```
做 Granger 因果檢定，檢驗：
1. 匯率是否 Granger-cause 股市報酬
2. 利率是否 Granger-cause 股市報酬
3. 股市報酬是否 Granger-cause 匯率
顯示 F-statistic 和 p-value
```

（執行）

> Granger 因果不是真正的因果關係——它只是說「一個變數的過去值能不能幫助預測另一個變數」。但在時間序列分析裡，這已經是很有用的資訊了。

**【Step 4：衝擊反應函數】**

```
畫出衝擊反應函數（IRF），
觀察「匯率衝擊」對「股市報酬」的影響持續多久，
包含 95% 信賴區間，期間設定 20 期
```

（執行）

> IRF 告訴你：如果匯率突然貶值一個標準差，股市會怎麼反應？反應多久？多少期之後回到零？這張圖也可以直接放進論文。

**【Step 5：變異數分解】**

```
做預測誤差變異數分解（FEVD），期間 20 期，
看股市報酬的預測誤差中，有多少比例來自匯率和利率的衝擊
```

（執行）

> 如果匯率解釋了股市報酬變異的 15%，那代表匯率對股市有實質的影響力。這種量化的結論，比「匯率對股市有影響」有說服力多了。

### 3-1. 驗證 AI 生成程式碼的正確性（15min）

**【重點提示】**

- **方法 2：檢查統計性質**
  - 自由度對不對？

- **方法 3：用 AI 自我檢查**

```

```

> - 報酬率用簡單報酬率而非對數報酬率
> - 忘記對資料排序（merge 之後順序亂掉）
